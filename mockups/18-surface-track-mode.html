Reference: Surface View -- Track Mode
Track Mode
  • Per-track surface showing AI-proposed performance controls
  • AI analyzes the track's chain and proposes meaningful macro controls
  • Human can edit, rearrange, add, or remove modules from the surface
  • Surface persists per-track -- switching tracks shows that track's surface
Module Types Shown
  • Live Pattern: read-only 16-step visualization of Marbles output (gate + pitch)
  • Knob Group: cluster of related rotary controls (Character: 3 knobs, Tone: 3 knobs)
  • XY Pad: 2D control space mapping two parameters (Brightness x Movement)
  • Chain Strip: compact signal flow overview with bypass toggles and claim indicators
Patch Behind This Surface
  • 6 modules: Plaits (VA source), Marbles Voltage (pitch sequences), Marbles Gate (trigger patterns), Ripples (filter), Tides (LFO), Compressor (sidechained to kick)
  • ~30 raw parameters across all modules
  • Surface distills to 7 meaningful controls: Density, Chaos, Memory, Brightness, Movement, Depth, plus the XY pad
  • Pattern module is read-only visualization, not a control
Mode Toggle & Customization
  • Track / Project toggle: Track mode shows per-track surface, Project mode shows cross-track overview
  • Same pill-toggle visual treatment as Pat/Song in the transport
  • "+" button adds new modules to the surface
  • Customize by: dragging modules to reorder, asking AI in Chat to redesign, or removing unwanted controls
  • Claim (hand icon) on chain modules protects them from AI modification
