Surface Palette System

How the AI generates a coherent color palette for any track's Surface from its base color.

Identity
Which track am I looking at? The base color answers this.
Function
What kind of control is this? Accent roles answer this.
Hierarchy
What's primary vs secondary? Saturation and brightness answer this.
Influenced by Material Design 3
Like MD3, a single seed color generates a full scheme via algorithmic hue rotation, and tonal palettes provide hierarchy within each role. But our roles are functional -- tied to the purpose of the control (generative, tonal, spatial) rather than generic container semantics (primary, secondary, tertiary).

Desaturation Rule

The base color is the most vivid thing on screen -- it is the track's identity, the anchor. Accent roles (Generative, Tonal, Spatial) should be slightly less saturated than the base to avoid visual competition. The base wins the saturation contest.

Separation Rule

When two roles' derived hues land within 30 degrees of each other, push them apart until the minimum gap is restored. The system guarantees at least 30 degrees of hue separation between all non-neutral roles.

Application Rules

Module to Role Mapping

Module / Element Color Role Reason
Pattern viz (steps) Base Shows the track's output
Knob Group (Marbles params) Generative Controls algorithmic / random behavior
Knob Group (filter / timbre) Tonal Shapes the sound character
XY Pad cursor Spatial Exploration / navigation control
Chain strip: source Base The track's fundamental identity
Chain strip: processors Tonal or Neutral Musical vs utility role
Chain strip: modulators Generative Modulation sources
Kill switches (Perf mode) Section Base Section identity
Filter knobs (Perf mode) Tonal Timbral shaping
Global gesture sliders Own fixed colors Energy=emerald, Tension=amber, Scene=violet

What NOT to Color

x Module card backgrounds -- always bg-surface
x Text labels -- always text-secondary or text-muted
x Module borders -- border-subtle, optional 15% role tint
x Footer, topbar, sidebar -- never affected by palette
x Canonical views (Rack, Patch, Tracker) -- deliberately neutral, structural
x Chat tab -- conversation, not an instrument panel

The Surface palette is Surface-specific. Canonical views stay precise, sparse, and neutral -- the contrast between Surface (expressive) and Rack/Patch/Tracker (x-ray) is intentional. The sidebar track accent dots are the connection point: the same colors that identify tracks in the sidebar become the seed colors for their Surface palettes.
